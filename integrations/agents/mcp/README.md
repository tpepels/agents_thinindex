# MCP Integration Deferred

thinindex does not currently bundle an MCP server, MCP client configuration, or MCP helper command. Normal thinindex use is through the local CLI commands and repo-local instruction files generated by `wi-init`.

This deferral is intentional: the current minimum useful agent integration is repo-local config helper support, including `wi-init --dry-run`. An MCP server would add another tool surface that needs repo path validation, bounded output handling, stale-index parity with `wi`, and client-specific setup that is broader than the current release-readiness scope.

A future MCP or tool wrapper must:

- expose `build_index`, `wi`, `wi refs`, `wi pack`, `wi impact`, and `wi-stats` as local commands;
- preserve `wi --help` as the source of truth for syntax and filters;
- validate repository paths before serving results;
- keep outputs bounded and compact for agents;
- match `wi` missing/stale index self-healing;
- avoid arbitrary shell execution;
- avoid network access, telemetry, hosted state, or proprietary agent APIs;
- avoid quality/comparator/real-repo workflows in normal search calls;
- record only the existing repository-local usage events;
- surface the same advisory limitation as `wi-stats`: local usage cannot prove that an agent avoided external grep, find, ls, or file reads.

This file is advisory documentation only. No MCP server is implemented or bundled.
